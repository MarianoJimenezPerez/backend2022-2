El servidor montado en el root de este proyecto escucha por defecto el puerto 8080 en modo cluster
Para poder montar un servidor cluster o fork (diferenciando por parámetros) ubicarse en la carpeta utils y correr el content.js enviando como param1 el puerto y como param 2 el modo

ejemplo:
    node content.js 8085 fork //esto iniciaría el servidor en el puerto 8085 en modo fork
